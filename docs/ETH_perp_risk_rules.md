## 2. 杠杆与仓位规则

### 2.1 杠杆倍数

- **固定使用 5x 杠杆** 进行 ETHUSDT 永续合约交易。

理论上：

- 单笔 TP 对资金的收益 ≈ `+1.75% × 5 = +8.75%`
- 单笔 SL 对资金的亏损 ≈ `-0.9% × 5 = -4.5%`

> 实际结果会因为资金利用率、手续费、滑点等略有偏差，以上为近似参考。

### 2.2 策略专用资金与单笔名义仓位

- 为该策略单独划分一笔“策略专用资金”`F_strategy`（例如 `10,000 USDT`）。
- **单笔名义仓位**设为：

  ```text
  N = α × F_strategy
  ```

  其中：

  - 初始建议：`α = 0.30`（30%）
  - 最大不超过：`α = 0.50`（50%）

示例（`F_strategy = 10,000 USDT`）：

- 初始名义仓位 `N = 3,000 USDT`；
- 在 5x 杠杆下，占用保证金 ≈ `3,000 / 5 = 600 USDT`。

单笔触发 SL 的资金亏损近似为：

- 标的亏损 `-0.9%` × 杠杆 `5x` ≈ `-4.5%`（相对名义仓位）；
- 对策略资金影响 ≈ `0.045 × 0.30 = 1.35%`（以 30% 仓位比例计）。

## 3. 单仓规则（硬性规则 A）

> **任何时刻，只允许持有一笔 ETH 合约仓位（LONG 或 SHORT）。**  
> 不允许多个同方向或对冲仓位叠加。

执行原则：

- 在存在未平仓仓位的情况下，**忽略所有新信号**；
- 只有在上一笔仓位完全平仓后，才再次接受新信号并开仓。

此规则与离线回测中的 `position-mode=single` 保持一致。

---

## 4. 连续亏损熔断（硬性规则）

> **最多连续 3 笔亏损。**  
> 当出现连续 3 笔亏损时，立即暂停该策略新开仓，由人工复盘后再决定是否恢复。

定义：

- 一笔交易的 PnL（包含手续费） < 0 则视为“亏损”；
- 连续亏损次数 `consec_losses` 为最近交易中，连续亏损的笔数。

规则：

1. 每当一笔 ETH 仓位平仓时：
   - 若 `PnL < 0` → `consec_losses += 1`；
   - 否则 → `consec_losses = 0`。
2. 若 `consec_losses >= 3`：
   - 将状态 `trading_paused` 置为 `true`；
   - 暂停该策略一切新开仓行为；
   - 触发告警（例如发送通知到监控/IM 群）。

恢复流程（人工）：

- 复盘最近亏损交易，确认模型/市场是否出现结构性问题；
- 如确认可继续运行，手动：
  - 重置 `consec_losses = 0`；
  - 设置 `trading_paused = false`；
- 策略恢复接收新信号。

## 5. 推荐的附加限制（可选）

以下限制不属于你当前明确要求，但建议一起考虑：

1. **24 小时最大开仓次数**  
   - 建议：24 小时内最多开 **3 笔** ETH 合约仓位。
   - 目的：防止在异常行情/模型失效时频繁交易。

2. **单日 / 单周最大资金回撤**  
   - 可选限制，例如：
     - 当日累计亏损 ≤ `3% × F_strategy`；
     - 当周累计亏损 ≤ `6% × F_strategy`。
   - 达到任一条件时，暂停该策略新开仓，由人工决定是否恢复。

结合 event_v3 1h 策略在 ETH 上的表现（高胜率、低频、带多周期过滤），本风控方案为：

- **5x ETH 永续杠杆**；

在不改变信号生成逻辑的前提下，这些规则为实盘合约提供了清晰的风险边界。
